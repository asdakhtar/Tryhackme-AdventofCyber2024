## 🎯 Advent of Cyber – Day 13: WebSockets Hijinks

### 🕸️ Topic: WebSockets Exploitation

Today’s challenge focused on identifying and manipulating WebSocket communication to gain unauthorized access to sensitive data — a common theme in modern web apps.

### 🛠️ Lab Flow
- Accessed a vehicle tracking web app with a “Track Vehicle” button.
- Set up **Burp Suite** to intercept WebSocket traffic.
- Upon clicking “Track,” the browser initiated WebSocket messages containing a `userid` parameter.
- Modified `userid` to reference the **mayor’s account**, e.g.:
  ```json
  {"userid":"8"}(its mayor’s user-id)
  ```
- WebSocket server responded with restricted data, including the **flag**.

### 🧩 Flags Discovered

| Flag # | Prompt                        | Value                                      |
|--------|-------------------------------|--------------------------------------------|
| 1️⃣     | What is the value of Flag1?   | `THM{dude_where_is_my_car}`                |
| 2️⃣     | What is the value of Flag2?   | `THM{my_name_is_malware._mayor_malware}`   |
